# Android App Icon Setup Guide

This guide explains how to set up app icons for Android according to Amazon Appstore guidelines.

## Requirements

### Amazon Appstore Requirements
- **512x512px PNG** icon for Appstore submission
- **No transparency** in the Appstore icon (solid background)
- Icons should be recognizable at small sizes

### Android Requirements
- **Standard launcher icons** in multiple densities:
  - mdpi: 48x48px
  - hdpi: 72x72px
  - xhdpi: 96x96px
  - xxhdpi: 144x144px
  - xxxhdpi: 192x192px

- **Adaptive icons** (Android 8.0+):
  - Foreground: 108dp (safe zone 72dp)
  - Background: 108dp (can be solid color)
  - Monochrome: 108dp (for Android 13+)

## Setup Steps

### 1. Prepare Source Icon

1. Create a **512x512px PNG** version of your app icon
2. Save it to: `assets/app-icons/bluefly-icon-512.png`
3. The icon should:
   - Be square (512x512px)
   - Have transparent background (for foreground layer)
   - Be centered with some padding (safe zone)

### 2. Generate All Icon Sizes

Run the icon generation script:

```bash
./scripts/generate-android-icons.sh assets/app-icons/bluefly-icon-512.png
```

This will automatically generate:
- Standard launcher icons for all densities
- Adaptive icon foregrounds
- Monochrome icons

### 3. Verify Generated Icons

Check that icons were created in:
```
android/app/src/main/res/
├── mipmap-mdpi/
│   ├── ic_launcher.png
│   ├── ic_launcher_foreground.png
│   └── ic_launcher_monochrome.png
├── mipmap-hdpi/
│   └── ...
├── mipmap-xhdpi/
│   └── ...
├── mipmap-xxhdpi/
│   └── ...
└── mipmap-xxxhdpi/
    └── ...
```

### 4. Update app.config.js (if needed)

The `app.config.js` already references:
```javascript
android: {
  icon: './assets/app-icons/bluefly-logo-lg.png',
  adaptiveIcon: {
    foregroundImage: './assets/app-icons/bluefly-logo-lg.png',
    backgroundColor: '#0085FF',
  },
}
```

If you want to use a different source icon, update these paths.

### 5. Rebuild the App

After generating icons, rebuild the app:

```bash
# Clean and rebuild
cd android
./gradlew clean
./gradlew assembleRelease
```

## Icon Specifications

### Standard Launcher Icons
- Format: PNG
- No transparency (will be placed on system background)
- Square format
- Sizes: 48px, 72px, 96px, 144px, 192px

### Adaptive Icons
- **Foreground**: 
  - Size: 108dp (432px for xxxhdpi)
  - Safe zone: 72dp (keep important content within this area)
  - Can have transparency
  - Will be masked by system
  
- **Background**:
  - Size: 108dp
  - Solid color (defined in `values/colors.xml` as `iconBackground`)
  - Current color: `#0085FF` (blue)

- **Monochrome**:
  - Size: 108dp
  - Used for Android 13+ themed icons
  - Should be grayscale/black and white

## Amazon Appstore Submission

For Amazon Appstore submission, you'll need:
1. **512x512px PNG** icon (no transparency)
2. All density icons (generated by script)
3. Adaptive icon configuration (already set up)

The 512x512px icon will be used as the Appstore listing icon.

## Troubleshooting

### Icons not showing correctly
- Verify all density folders have icons
- Check that adaptive icon XML exists: `mipmap-anydpi-v26/ic_launcher.xml`
- Ensure background color is set in `values/colors.xml`

### Icons look blurry
- Make sure source icon is at least 512x512px
- Verify all density sizes are correct
- Check that icons are PNG format (not WebP)

### Monochrome icons not working
- Install ImageMagick for true grayscale conversion:
  ```bash
  brew install imagemagick
  ```
- Then convert manually:
  ```bash
  convert icon.png -colorspace Gray -colors 2 monochrome.png
  ```
